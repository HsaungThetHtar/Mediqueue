# คู่มือการใช้งาน MediQueue แยกตาม Role

เอกสารนี้อธิบายขั้นตอนการใช้งานระบบสำหรับแต่ละบทบาท (ผู้ป่วย, แอดมิน, แพทย์) ตามลำดับการทำงานจริง

---

## 1. ผู้ป่วย (Patient)

### 1.1 ลงทะเบียนและเข้าสู่ระบบ

1. เปิดเว็บแล้วไปที่หน้า **Sign In** (หรือ `/signup` สำหรับสมัครใหม่)
2. **สมัครสมาชิก (Sign Up)**
   - กรอก: ชื่อ-นามสกุล, อีเมล, เบอร์โทร, เพศ, วันเกิด, เลขบัตรประชาชน, รหัสผ่าน และยืนยันรหัสผ่าน
   - กด **Sign Up** — ระบบจะล็อกอินให้และพาไป flow จอง (หรือแดชบอร์ด)
3. **เข้าสู่ระบบ (Sign In)**
   - กรอกอีเมล (หรือเบอร์โทร) และรหัสผ่าน
   - กด **Sign In** — ระบบจะพาไปตาม role (ผู้ป่วย → หน้าเลือกวันที่/แผนก, แอดมิน → /admin, หมอ → /doctor)

### 1.2 จองคิว

1. หลังล็อกอินเป็นผู้ป่วย จะเข้าหน้า **เลือกวันที่และแผนก** (`/app/select-date`)
   - เลือกวันที่นัด
   - เลือกแผนก (เช่น Internal Medicine, Pediatrics)
   - กดต่อไป
2. **เลือกหมอ** (`/app/select-doctor`)
   - เลือกหมอจากรายการ (แสดงจำนวนคิวต่อวัน)
   - กดต่อไป
3. **เลือกช่วงเวลาและยืนยันจอง** (`/app/booking`)
   - เลือกช่วง: **Morning (08.00–12.00)** หรือ **Afternoon (13.00–17.00)**
   - ดูจำนวนคิวที่เหลือ (เช่น 6/15)
   - กด **Confirm Booking**
   - **ต้องล็อกอินอยู่** — ถ้ายังไม่ล็อกอินระบบจะแจ้งให้เข้าสู่ระบบก่อน
4. หลังจองสำเร็จ จะถูกพาไปหน้า **ใบจอง (Slip)** (`/app/slip`)
   - แสดงหมายเลขคิว, หมอ, วันที่, เวลาประมาณ, กำลังรับบริการหมายเลขใด, เวลารอโดยประมาณ (อัปเดตจากระบบ)

### 1.3 เช็คอิน (ก่อนพบหมอ)

1. เข้า **Check-In** ได้จากเมนูหรือลิงก์ไป `/check-in`
2. **กรณีมี booking อยู่แล้ว (เช่น จากแดชบอร์ด)**
   - ระบบอาจส่งข้อมูลการจองมาพร้อมหน้า → แสดงข้อมูลและกดยืนยันเช็คอินได้เลย
3. **กรณีใส่รหัสมือ**
   - กรอกหมายเลขคิว (เช่น M-001) หรือรหัสการจอง
   - กด **Verify Code** — ระบบตรวจจาก server แล้วแสดงข้อมูลจอง
   - ตรวจสอบข้อมูลแล้วกด **Confirm Check-In**
4. หลังเช็คอินสำเร็จ จะแสดงข้อความสำเร็จและหมายเลขคิว

### 1.4 แดชบอร์ดผู้ป่วย

1. ไปที่ **Dashboard** (`/dashboard`)
2. แสดง:
   - การจองที่กำลังจะมาถึงและที่ผ่านมา
   - สถานะ (Upcoming, Checked In, In Progress, Completed, Cancelled)
   - เวลารอโดยประมาณ (นาที) จากระบบ
3. กดรายการเพื่อดูรายละเอียด; ถ้ามีนัดที่ยังไม่เช็คอิน สามารถไปหน้า Check-In ได้

### 1.5 การแจ้งเตือน

1. ไปที่ **Notifications** (`/notifications`)
2. ดูรายการแจ้งเตือน (จอง, สถานะ, เตือน ฯลฯ)
3. กดเครื่องหมายอ่านแล้ว / ลบรายการ / ลบทั้งหมด ได้ (การลบบันทึกในระบบ)

### 1.6 ตั้งค่าโปรไฟล์

1. ไปที่ **Settings** (`/settings`)
2. แก้ไข: ชื่อ, เบอร์โทร, วันเกิด, เลขบัตร
3. เปลี่ยนรหัสผ่าน (กรอกรหัสเดิมและรหัสใหม่)
4. กดบันทึก — ข้อมูลส่งไปยัง server

## 2. แอดมิน (Admin)

### 2.1 เข้าสู่ระบบ

1. เปิดหน้า **Sign In** (`/` หรือ `/signin`)
2. กรอกอีเมลและรหัสผ่านของ **บัญชีแอดมิน** (เช่น หลัง seed: `admin@mediqueue.com` / `password123`)
3. กด **Sign In** — ระบบจะพาไป **Admin Dashboard** (`/admin` หรือ `/admin/dashboard`)

### 2.2 แดชบอร์ด (Dashboard)

1. หน้าแรกแอดมินแสดง:
   - สถิติวันนี้: Total Bookings, Checked-in, Waiting, In Progress, Completed (และเทรนด์เทียบวันก่อน)
   - ตารางคิว **Morning** และ **Afternoon** แยกตามช่วง
   - ปุ่ม **View All Queues** ไปหน้า Queue Management
2. เลือกวันที่ (filter วันที่) เพื่อดูข้อมูลวันอื่น
3. ถ้ามีหลายหน้า ใช้ปุ่ม pagination (Prev, หมายเลขหน้า, Next) ด้านล่างตาราง

### 2.3 จัดการคิว (Queue Management)

1. ไปที่ **Queue** (`/admin/queue`)
2. แสดงรายการคิว (ตามวันที่ที่เลือก) แยกช่วงเช้า/บ่าย
3. การกระทำต่อแถวคิว:
   - **Call** — เรียกคนไข้ (เปลี่ยนสถานะเป็น in-progress)
   - **Skip** — ข้ามคิว (ยกเลิกรายการนั้น อาจเลื่อนคิวถัดไป)
   - **Complete** — จบการรับบริการ (completed)
4. แก้สถานะผ่าน dropdown หรือ modal ตาม UI (เช่น Waiting → Checked-in, Cancelled ฯลฯ)

### 2.4 รายการรอ (Waiting List)

1. ไปที่ **Waiting** (`/admin/waiting`)
2. ดูรายการที่รอ/ยืนยัน/เช็คอินแล้ว
3. เปลี่ยนสถานะหรือยกเลิกการจองได้ (ใช้ API อัปเดตสถานะ)

### 2.5 รายการจอง (Bookings)

1. ไปที่ **Bookings** (`/admin/bookings`)
2. ดูรายการจองทั้งหมด (มี filter วันที่/แผนก/สถานะ ตาม UI)
3. กดเข้า **รายละเอียดจอง** (Booking Details) เพื่อ:
   - เปลี่ยนสถานะ
   - แก้ไขหมอ/วันที่/ช่วงเวลา
   - ยกเลิกการจอง (พร้อมเหตุผล)
   - เช็คอินให้คนไข้ (manual check-in)

### 2.6 Check-Ins

1. ไปที่ **Check-Ins** (`/admin/checkins`)
2. เลือกวันที่
3. ดูรายการเช็คอิน (QR / manual)
4. ปุ่ม **เช็คอินมือ** สำหรับการจองที่ยังไม่เช็คอิน — กดแล้วระบบอัปเดตสถานะและสร้างการแจ้งเตือนให้คนไข้

### 2.7 Completed

1. ไปที่ **Completed** (`/admin/completed`)
2. ดูรายการนัดที่เสร็จแล้ว (สถานะ completed) ตามวันที่

### 2.8 ตั้งค่า (Settings)

1. ไปที่ **Settings** (`/admin/settings`)
2. **ตั้งค่าระบบ**
   - ชื่อโรงพยาบาล, จำนวนคิวต่อช่วง (เช้า/บ่าย), จำนวนคิวมัธยัสถ์ต่อวัน, เวลาทำการ
   - กดบันทึก — บันทึกลงฐานข้อมูล
3. **บัญชีผู้ใช้**
   - แก้ชื่อที่แสดง, อีเมล (อ่านอย่างเดียว)
   - บันทึกแล้วอัปเดตในระบบและใน Header
4. **การแจ้งเตือน**
   - เปิด/ปิดการแจ้งเตือน (จองใหม่, เรียกคิวเร่งด่วน, SMS ฯลฯ) — บันทึกลง User
5. **แผนก**
   - เพิ่ม/แก้ไข/ลบแผนก (เรียงลำดับการแสดง)
6. **หมอ**
   - เพิ่มหมอ (ชื่อ, แผนก, เวลาทำการ), แก้ไข, ลบ (ถ้าไม่มี booking ค้าง)

## 3. แพทย์ (Doctor)

### 3.1 เข้าสู่ระบบ

1. เปิดหน้า **Sign In**
2. กรอกอีเมลและรหัสผ่านของ **บัญชีหมอ** (หลัง seed เช่น `doctor@mediqueue.com` / `password123`)
3. กด **Sign In** — ระบบจะพาไป **Doctor Dashboard** (`/doctor`)

### 3.2 หน้า Doctor Dashboard

1. แสดง:
   - ชื่อหมอและวันที่
   - รายชื่อคนไข้วันนี้แบ่งเป็น: **รอรับบริการ**, **กำลังตรวจ**, **เสร็จแล้ว**
2. แต่ละรายการแสดง: หมายเลขคิว, ชื่อ, อายุ, เพศ, เวลาเช็คอิน (ข้อมูลจากระบบ)

### 3.3 เรียกคนไข้เข้าห้อง

1. กด **เรียกคิว** หรือปุ่มที่เกี่ยวข้องกับรายการคนไข้ที่รอ
2. ระบบเปลี่ยนสถานะเป็น **in-progress** และส่งการแจ้งเตือนให้คนไข้
3. คนไข้จะแสดงในส่วน "กำลังตรวจ"

### 3.4 หมายเหตุทางการแพทย์ (Medical Notes)

1. ขณะมีคนไข้ที่กำลังตรวจ เลือก **Add Notes** (หรือปุ่มแสดงหมายเหตุ)
2. กรอกข้อความในช่อง Consultation Notes
3. กด **Save Notes** — ระบบบันทึกลงการจอง (doctorNotes) และโหลดข้อมูลใหม่

### 3.5 จบการตรวจ (Complete Consultation)

1. เลือกคนไข้ที่กำลังตรวจอยู่
2. กด **Complete Consultation**
3. ระบบเปลี่ยนสถานะเป็น **completed**, ส่งการแจ้งเตือนให้คนไข้ และอาจเลื่อนคิวถัดไปให้เป็น in-progress อัตโนมัติ (ตาม logic backend)

## 4. สรุปเส้นทางหลัก (Routes) ตาม Role

| Role   | หน้า/เส้นทางหลัก |
|--------|-------------------|
| ทุกคน  | `/`, `/signin`, `/signup` |
| Patient| `/app/select-date`, `/app/select-doctor`, `/app/booking`, `/app/slip`, `/dashboard`, `/check-in`, `/notifications`, `/settings` |
| Admin  | `/admin`, `/admin/dashboard`, `/admin/queue`, `/admin/waiting`, `/admin/bookings`, `/admin/bookings/:id`, `/admin/checkins`, `/admin/completed`, `/admin/settings`, `/admin/profile` |
| Doctor | `/doctor` |

หลังล็อกอิน ระบบจะ redirect ตาม role ไปยังแดชบอร์ดหรือ flow ที่เหมาะสมให้โดยอัตโนมัติ
